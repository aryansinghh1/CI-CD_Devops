Infrastructure Layers

Node.js + Express.js - Application servers running containerized in Kubernetes pods
Docker + containerd - Container images built from multi-stage Dockerfile using Alpine Linux base
Kubernetes 1.29 - Pod management, service discovery, load balancing, and auto-scaling
Calico + CoreDNS - Pod-to-pod networking, service mesh, and DNS resolution
AWS EC2 - 8GB RAM, 50GB storage Ubuntu instance running Kubernetes control plane and worker nodes
Jenkins + GitHub - Automated builds triggered by webhook, orchestrating the entire deployment flow
Prometheus + Grafana - Metrics collection, storage, and visualization with real-time dashboards and alerts

💡 Key Design Decision: Using containerd instead of Docker daemon for Kubernetes provides better resource efficiency, is the industry standard for K8s production, and eliminates the Docker overhead layer while maintaining full container compatibility.

CI/CD Pipeline Detailed

Pipeline Stages & Workflow

Stage Description Tools Duration
1. Code Commit Developer pushes code to GitHub repository Git, GitHub Instant
2. Webhook Trigger GitHub webhook automatically triggers Jenkins GitHub, Jenkins <1s
3. Code Clone Jenkins clones repository to workspace Git, Jenkins ~10-15s
4. Build Application Compile and prepare application Node.js, npm ~20-30s
5. Run Tests Execute automated unit and integration tests Jest, Mocha, npm ~30-45s
6. Code Quality Check Analyze code for quality metrics and standards SonarQube, ESLint ~15-20s
7. Security Scan Scan dependencies and code for vulnerabilities npm audit, OWASP ~20-30s
8. Build Docker Image Create Docker image from Dockerfile Docker, Alpine Linux ~40-60s
9. Push to Registry Push image to Docker Hub with build number tag Docker Hub, jenkins ~20-40s
10. Deploy to K8s Update deployment manifests and apply to cluster kubectl, Kubernetes ~30-60s
11. Health Checks Verify container health and endpoint availability HEALTHCHECK, kubectl ~10-20s
12. Monitoring Setup ServiceMonitor deployment for Prometheus scraping Prometheus, Grafana ~5-10s

Jenkins Pipeline Configuration

pipeline { agent any environment { IMAGE_NAME = "aryansinghh1/devops-project" TAG = "${BUILD_NUMBER}" } stages { stage('Clone Code') { steps { git 'https://github.com/aryansinghh1/CI-CD_Devops.git' } } stage('Build Docker Image') { steps { sh 'docker build -t $IMAGE_NAME:$TAG .' } } stage('Push to DockerHub') { steps { sh 'docker push $IMAGE_NAME:$TAG' } } stage('Deploy to Kubernetes') { steps { sh ''' sed -i "s|image: .*|image: $IMAGE_NAME:$TAG|g" deployment.yaml kubectl apply -f deployment.yaml kubectl apply -f service.yaml kubectl apply -f servicemonitor.yaml ''' } } } }

Deployment & Setup Guide

Quick Start Steps

  1. EC2 Instance Setup: Create Ubuntu 22.04 LTS instance with 8GB RAM, 50GB storage on AWS
  2. System Update: Run apt update && apt upgrade to install latest packages
  3. Docker Installation: Install Docker and add user to docker group
  4. Kubernetes Setup: Install kubeadm, kubelet, kubectl; initialize cluster with kubeadm init
  5. Network Plugin: Deploy Calico for pod networking with kubectl apply
  6. Containerd Configuration: Set SystemdCgroup = true in /etc/containerd/config.toml
  7. Jenkins Installation: Install Java 21 JRE, then Jenkins from official repository
  8. Jenkins Configuration: Setup GitHub webhooks, Docker plugin, Kubernetes CLI plugin
  9. Monitoring Stack: Use Helm to install Prometheus community kube-prometheus-stack
  10. Application Deployment: Apply deployment.yaml, service.yaml, servicemonitor.yaml to K8s

📚 Complete Setup Guide: Refer to README.md in the repository for detailed step-by-step instructions with all commands and configurations.

Key Security Configurations

🔐 Container Security

Non-root user (nodejs:1001) runs the application instead of root. This limits damage if container is compromised.

  • User UID: 1001
  • Group GID: 1001
  • Read-only filesystem where possible
  • Minimal attack surface

🛡️ Network Security

AWS security groups restrict access to only required ports. Calico network policies provide micro-segmentation.

  • SSH (22) for admin only
  • Jenkins (8080) for CI/CD
  • App (3000) for users
  • Prometheus (9090) internal

⚙️ RBAC & Access Control

Kubernetes RBAC ensures Jenkins has only necessary permissions. Service accounts are properly configured.

  • ServiceAccount for Jenkins
  • ClusterRole with deploy permissions
  • kubeconfig security
  • Token-based access

📊 Health & Liveness

Docker HEALTHCHECK ensures only healthy containers serve traffic. Kubernetes liveness probes restart failed pods.

  • 30s check interval
  • 3s timeout
  • 5s startup grace
  • 3 retry attempts

Common Issues & Troubleshooting

❌ Kubernetes Nodes Not Ready

Cause: containerd service not running or misconfigured

Solution: Check containerd status, verify config.toml has SystemdCgroup = true

❌ Jenkins Cannot Access Kubernetes

Cause: Missing kubeconfig or incorrect permissions

Solution: Verify /var/lib/jenkins/.kube/config exists with correct owner

❌ Docker Push Fails

Cause: Not logged into Docker Hub or incorrect image name

Solution: Run docker login and verify image name format

❌ Monitoring Shows No Data

Cause: ServiceMonitor labels don't match deployment labels

Solution: Check and align label selectors in deployment and servicemonitor

❌ Pod CrashLoopBackOff

Cause: Application startup failure or resource limits exceeded

Solution: Check logs with kubectl logs POD_NAME, verify resources

❌ Webhook Not Triggering

Cause: Webhook URL incorrect or Jenkins not accessible

Solution: Verify Jenkins URL is reachable, check webhook delivery logs on GitHub

DevOps Best Practices Implemented

✅ Infrastructure as Code (IaC)

All infrastructure defined in code: Terraform for EC2, Kubernetes manifests for deployments, Helm charts for monitoring stack.

✅ Continuous Monitoring

24/7 monitoring with Prometheus and Grafana. Application metrics exposed via /metrics endpoint. Real-time dashboards and alerts.

✅ Automated Testing

Jenkins pipeline includes code quality checks, security scanning, and automated tests before deployment.

✅ Container Best Practices

Multi-stage builds, Alpine Linux base, non-root user, health checks, minimal image size, fast deployment.

✅ Immutable Infrastructure

Containers are immutable, reproducible deployments, easy rollbacks, zero-downtime deployments with rolling updates.

✅ Version Control Everything

All code and configuration in Git. Enables full audit trail, easy rollback, and collaborative development.
